# Mail Check Skill

## 개요
이 Skill은 사용자의 메일함을 확인하고, 중요도 기준으로 필터링한 뒤,
핵심 내용을 요약하는 절차적 지식(Procedural Knowledge)을 정의합니다.

Skill은 Agent에게 "어떤 순서로, 어떤 판단 기준으로" 작업을 수행할지 가르치는
**절차적 지시서**입니다. MCP Tool이 "무엇을 할 수 있는가"(능력)를 제공한다면,
Skill은 "어떻게 해야 하는가"(전략)를 제공합니다.

## When to Use (언제 이 Skill을 활성화하는가)
다음과 같은 사용자 요청이 들어왔을 때 이 Skill을 사용합니다:
- "메일 확인해줘", "받은 편지함 좀 봐줘"
- "중요한 메일만 골라줘", "긴급 메일 있어?"
- "오늘 온 메일 요약해줘"
- "메일 검색해줘", "OO한테 온 메일 찾아줘"

## 사전 조건 (Prerequisites)
- MCP 서버 `mail-mcp-server`가 연결되어 있어야 합니다.
- 다음 MCP Tool들이 사용 가능해야 합니다:
  - `check_inbox` : 메일함 목록 조회
  - `get_email_detail` : 개별 메일 상세 조회
  - `search_emails` : 메일 검색

## Instructions (실행 절차)

### Step 1: 메일함 확인
1. `check_inbox` MCP Tool을 호출하여 메일 목록을 가져옵니다.
2. 사용자가 특정 필터를 요청한 경우 해당 필터를 적용합니다:
   - "중요한 메일" -> filter: "important"
   - "안 읽은 메일" -> filter: "unread"
   - "모든 메일"   -> filter: "all"
3. 필터 지정이 없으면 기본값 `"unread"`를 사용합니다.

### Step 2: 중요도 판단 및 필터링
메일 목록을 받은 후 다음 기준으로 중요도를 판단합니다:

**높은 중요도 (즉시 보고):**
- 제목에 "긴급", "urgent", "ASAP" 포함
- 발신자가 "팀장", "대표", "고객사"인 경우
- 마감 기한이 24시간 이내인 경우

**보통 중요도 (요약에 포함):**
- 프로젝트 관련 업데이트
- 회의 일정 관련

**낮은 중요도 (별도 언급 없이 개수만 보고):**
- 뉴스레터, 광고성 메일
- 자동 알림 (빌드 결과, 모니터링 등)

### Step 3: 상세 내용 확인 (필요시)
1. 높은 중요도 메일이 있으면, `get_email_detail`로 상세 내용을 확인합니다.
2. 사용자가 특정 메일의 상세 내용을 요청한 경우에도 호출합니다.
3. 한 번에 최대 3개까지만 상세 조회합니다 (너무 많은 Tool 호출 방지).

### Step 4: 검색 (사용자가 검색을 요청한 경우)
1. 사용자가 특정 키워드나 발신자로 검색을 요청하면 `search_emails`를 사용합니다.
2. 검색 결과가 없으면 사용자에게 알리고, 다른 키워드를 제안합니다.

### Step 5: 결과 요약 및 보고
최종 응답은 다음 형식을 따릅니다:

```
[메일 요약 보고]

긴급 (N건):
  - 발신자: 제목 - 핵심 내용 한 줄

일반 (N건):
  - 발신자: 제목

기타 N건은 중요도가 낮아 생략했습니다.

권장 조치:
  - (긴급 메일에 대해 필요한 액션 제안)
```

## 주의사항 (Guardrails)
- 메일 내용에 개인정보(주민번호, 카드번호 등)가 포함되어 있으면 마스킹합니다.
- 한 번의 요청에서 Tool 호출은 최대 5회로 제한합니다.
- 메일 삭제, 전달 등 쓰기 작업은 이 Skill의 범위가 아닙니다.
  사용자가 요청하면 "이 Skill은 읽기 전용입니다"라고 안내합니다.

## 예시 대화

**사용자:** "오늘 메일 확인해줘"
**Agent 동작:**
1. `check_inbox(filter="unread")` 호출
2. 중요도 분류
3. 긴급 메일 `get_email_detail()` 호출
4. 요약 보고

**사용자:** "고객사에서 온 메일 찾아줘"
**Agent 동작:**
1. `search_emails(query="고객사")` 호출
2. 결과 요약
