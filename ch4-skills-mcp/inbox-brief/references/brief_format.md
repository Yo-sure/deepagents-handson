# brief.md 형식 (점진 공개 — 2단계)

SKILL.md가 큰 틀을 정하고, 세부 형식은 이 파일에 둔다. 에이전트는 브리프를 실제로 쓸 때만
이 파일을 읽는다. 컨텍스트를 아끼는 progressive disclosure 패턴이다.

## 골격

```markdown
# 인박스 브리프 — 2026년 5월

## 한 줄 요약
문서 N건 · 영수증 지출 합계 X원 · 짚을 점 K건.

## 지출
- 식비: ...원
- 교통: ...원
- 생활: ...원

## 짚을 점
- (gap) 쿠팡 89,000원 — 영수증 없음, 확인 필요
- (subscription) 넷플릭스 17,000원 — 구독 추정

## 할 일
- [ ] 쿠팡 결제 영수증 확인
- [ ] 구독 목록 점검
```

## 규칙
- 한 화면(노트북 기준 한 스크롤)을 넘기지 않는다.
- 금액은 천 단위 구분, 원 단위.
- "짚을 점"은 knowledge_base의 gap·subscription 항목에서 1:1로 가져온다.
- 추정에는 "추정", 미확인에는 "확인 필요"를 분명히 단다.
